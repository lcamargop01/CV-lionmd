Module 5 — Access Control
User Management & Security
Control who can see what — down to the individual user.

Invite-Only Access

No self-registration. Admins create accounts and send a secure one-time invite link. Nobody gets in uninvited.

Forgot Password Flow

Providers can request a password reset by email. Reset link re-uses the secure invite flow — no new attack surface.

Role-Based Access

Admin, CareValidate, Onboarding, Provider, and Candidate roles each see only what they need.

Deactivate Users

Remove access instantly without deleting the user record. Re-enable if needed.

Reset Invite Links

Regenerate a fresh invite link for any user — invalidates the old one immediately for security.

Link Contractors to Users

Connect a portal user account to their contractor record so payroll data automatically appears in their provider view.

All passwords are hashed with PBKDF2-SHA256 (100,000 rounds). Tokens use HMAC-SHA256. No passwords are stored in plain text.
